## 项目主要需求

### 项目概述

【Web前端】供应链物流数据看板。基于 ECharts 的纯前端物流监控与分析平台，实时展示运输状态、仓库容量热力图、车队分布和供应链成本分析。数据存储于 localStorage / IndexedDB。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个数据模型）

1. **运输单（Shipment）** — 字段：运单号、发货仓库ID、收货地址、货物名称、重量（kg）、体积（m³）、运输方式（Road/Rail/Sea/Air）、状态（Pending/InTransit/Delivered/Exception/Returned）、发货时间、预计到达时间、实际到达时间、司机/承运人、当前位置坐标
   - 定义位置：`types/shipment.ts`

2. **仓库容量（WarehouseCapacity）** — 字段：仓库ID、仓库名称、总容量（m³）、已用容量、分区列表（分区名+当前占用率0-100）、经纬度坐标、区域（华东/华南/华北/西南/西北/东北/华中）
   - 定义位置：`types/warehouse.ts`

3. **车队信息（FleetVehicle）** — 字段：车牌号、车辆类型（轻卡/中卡/重卡/冷链车）、当前状态（Idle/OnRoute/Maintenance）、当前坐标、当前运单ID、累计里程、上次保养日期、下次保养日期
   - 定义位置：`types/fleet.ts`

4. **成本记录（CostEntry）** — 字段：所属运单ID、成本类型（Fuel/Toll/Labor/Maintenance/Insurance/Other）、金额、日期、备注
   - 定义位置：`types/cost.ts`

5. **物流节点（TrackingEvent）** — 字段：所属运单ID、节点类型（Departure/Transit/Arrival/Delay/Exception）、地点、时间、描述
   - 定义位置：`types/tracking.ts`

### 核心页面（≥ 4 个）

1. **物流总览仪表盘** `/dashboard` — 消费：Shipment + WarehouseCapacity + FleetVehicle
   - 今日在途运单数、延误预警、仓库容量TOP10柱状图、车辆状态环形图
   - 使用 `<StatCard>`, `<StatusRing>`, `<AlertBanner>` 共享组件

2. **运输监控** `/shipments` — 消费：Shipment + TrackingEvent
   - 运单列表（按状态Tab），运单详情（物流时间线+当前位置地图点）
   - 使用 `<StatusBadge>`, `<Timeline>`, `<EmptyState>`

3. **仓库分析** `/warehouse-analytics` — 消费：WarehouseCapacity
   - 热力图展示各仓库/各分区的容量占用率（颜色深浅表示），容量趋势折线图
   - 使用 `<HeatmapCell>`, `<StatCard>`

4. **车队追踪** `/fleet` — 消费：FleetVehicle + Shipment
   - 车辆列表（按状态筛选），地图标记所有车辆位置，车辆详情（里程趋势、保养记录）
   - 使用 `<StatusBadge>`, `<StatusRing>`

5. **成本分析** `/cost-analytics` — 消费：CostEntry + Shipment
   - 成本构成饼图、月度成本趋势、单位运输成本对比、异常成本高亮
   - 使用 `<StatCard>`, `<AlertBanner>`

### 横切关注点

1. **全局异常处理**
   - 触达文件层：数据加载失败fallback + 图表渲染错误捕获 + IndexedDB操作异常 + ErrorBoundary + 错误类型定义

2. **主题切换（深色/浅色模式）**
   - 触达文件层：CSS变量 + themeStore + 所有组件适配 + 图表主题切换（ECharts dark主题）+ 主题类型定义

### 共享枚举/常量（≥ 2 组）

1. **运输状态 ShipmentStatus**：Pending / InTransit / Delivered / Exception / Returned
   - 前端：`types/enums.ts`

2. **运输方式 TransportMode**：Road / Rail / Sea / Air

3. **车辆状态 VehicleStatus**：Idle / OnRoute / Maintenance

4. **成本类型 CostType**：Fuel / Toll / Labor / Maintenance / Insurance / Other

### 共享前端组件清单（≥ 3 个）

1. **`<StatusBadge>`** — 运输监控、车队追踪
2. **`<StatCard>`** — 仪表盘、仓库分析、成本分析
3. **`<AlertBanner>`** — 仪表盘（延误预警）、成本分析（异常成本）
4. **`<StatusRing>`** — 仪表盘、车队追踪（环形状态图）
5. **`<Timeline>`** — 运输监控（物流时间线）

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useMockData`** — 生成逼真的模拟物流数据（运单、仓库、车队）
2. **`useChartTheme`** — 根据当前主题返回对应的 ECharts 配置
3. **`formatDistance`** / **`formatWeight`** — 距离和重量格式化

### 技术要求

- **前端端口**：18706
- **前端**：React 18 + TypeScript + Vite
- **图表**：ECharts 5（折线、柱状、饼图、热力图、地图）
- **UI 库**：Tailwind CSS
- **状态管理**：Zustand
- **本地存储**：IndexedDB（Dexie.js）
- **路由**：React Router v6
- **构建工具**：Vite
